# Navix 桌面客户端

## 概述

Navix 桌面客户端位于 `apps/desktop`，基于 Tauri v2、React 19、TypeScript 与本地 SQLite 构建。

它提供导航面板、账户能力、服务端同步和基础应用设置等功能。

## 主要能力

### 导航面板

- 管理网站分组与网站项
- 调整组内排序
- 配置公网地址和局域网地址
- 抓取站点元数据与图标
- 导入浏览器书签
- 管理内置与自定义搜索引擎

### 账户与同步

- 支持匿名使用与多账户切换
- 支持登录、登出、用户名更新和本地账户数据清理
- 支持配置服务端地址并执行双向同步
- 同步流程包含兼容性检查、会话化分块上传、增量回传和图标传输

## 账号来源

- 服务端首次使用时，需要先完成管理员初始化。
- 普通用户账号由管理员在服务端管理界面创建。

### 应用设置

- 多语言
- 主题设置
- 开机自启
- 启动后最小化
- 关闭时最小化到托盘
- 自动更新检查

## 架构分层

桌面端代码主要分为两层：

- `apps/desktop/src`：React 界面、状态管理、页面与前端服务
- `apps/desktop/src-tauri`：Tauri 宿主、数据库、命令处理、托盘与本地能力

共享模块包括：

- `packages/shared-ts`
- `packages/shared-ui`
- `packages/shared-rs`

## 本地数据目录

客户端默认在用户目录下维护数据文件：

- 应用目录：`~/.vust/navix`
- 数据目录：`~/.vust/navix/data`
- 图标目录：`~/.vust/navix/icons`

本地数据库、配置与图标缓存围绕上述目录组织。

## 与服务端协作

- 客户端通过 `apps/server` 提供的认证、同步和图标接口与服务端协作。
- 首次同步前会进行版本与协议检查。
- 服务端实例标识会参与账号绑定与同步校验，切换服务端地址后可能需要重新登录。

## 开发与运行

在仓库根目录执行：

```bash
pnpm tauri dev
```

如需构建发布包：

```bash
pnpm build
```
